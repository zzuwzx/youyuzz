## 前置条件

### 必需软件

- **Python 3.10+** - [下载](https://www.python.org/downloads/)
- **Node.js 18+** - [下载](https://nodejs.org/)
- **Git** - [下载](https://git-scm.com/)

### Python 依赖

```powershell
pip install pyinstaller
pip install -r pc-client\backend\requirements.txt
```

### Node.js 依赖

```powershell
cd pc-client\frontend
npm install
```

### P2-M4 依赖

⚠️ **重要**: 本模块依赖 P2-M4（进程管理 Electron 化）完成。请确保以下文件存在：

- `pc-client/frontend/electron/main.ts`
- `pc-client/frontend/electron/pythonBridge.ts`
- `pc-client/frontend/electron/preload.ts`

## 快速开始

使用一键构建脚本：

```powershell
cd scripts
.\build.ps1
```

可选参数：

```powershell
.\build.ps1 -Clean              # 清理之前的构建产物
.\build.ps1 -SkipBackend        # 跳过后端打包
.\build.ps1 -SkipFrontend       # 跳过前端打包
```

构建产物：

- Python 后端: `pc-client\backend\dist\backend.exe`
- 安装包: `pc-client\frontend\release\鱿郁仔仔 Setup X.X.X.exe`

## 详细步骤

### 1. 准备应用图标

#### 图标规格

- **格式**: ICO
- **尺寸**: 16x16, 32x32, 48x48, 256x256（多尺寸合一）
- **位置**: 
  - `pc-client/backend/icon.ico` (PyInstaller)
  - `pc-client/frontend/assets/icon.ico` (Electron)

#### 制作方法

**方法 1: 在线工具**

1. 访问 https://convertio.co/png-ico/
2. 上传 PNG 图片（建议 256x256 以上）
3. 下载生成的 ICO 文件

**方法 2: ImageMagick 命令行**

```powershell
# 安装 ImageMagick
winget install ImageMagick.ImageMagick

# 转换 PNG 到 ICO
magick convert logo.png -resize 256x256 -define icon:auto-resize=16,32,48,256 icon.ico
```

**方法 3: Greenfish Icon Editor Pro（免费）**

1. 下载: https://greenfishsoftware.blogspot.com/p/greenfish-icon-editor-pro.html
2. 导入 PNG 图片
3. 添加多尺寸版本
4. 保存为 ICO

### 2. 打包 Python 后端

#### 步骤

```powershell
# 1. 进入后端目录
cd pc-client\backend

# 2. 激活虚拟环境（如果有）
venv\Scripts\Activate.ps1

# 3. 安装依赖
pip install -r requirements.txt

# 4. 运行 PyInstaller
pyinstaller build.spec --clean --noconfirm
```

#### 验证

```powershell
# 测试打包的程序
.\dist\backend.exe

# 应该看到 FastAPI 服务启动信息
# 访问 http://127.0.0.1:18888/docs 查看 API 文档
```

#### 常见问题

**问题**: `ModuleNotFoundError: No module named 'win32com'`

**解决**: 在 `build.spec` 中添加到 `hiddenimports`：

```python
hiddenimports=[
    'win32com',
    'win32com.client',
    # ... 其他导入
]
```

**问题**: 程序启动时报 DLL 错误

**解决**: 检查 `build.spec` 中的 `binaries` 配置，确保所有必要的 DLL 都已包含。

### 3. 打包 Electron 前端

#### 步骤

```powershell
# 1. 进入前端目录
cd pc-client\frontend

# 2. 安装依赖
npm install

# 3. 构建前端资源
npm run build

# 4. 打包 Electron 应用
npm run electron:build:win
```

#### 配置说明

编辑 `package.json` 中的 `build` 部分：

```json
{
  "build": {
    "appId": "com.youyuzz.pc-client",
    "productName": "鱿郁仔仔",
    "win": {
      "target": ["nsis"],
      "icon": "assets/icon.ico"
    },
    "nsis": {
      "allowToChangeInstallationDirectory": true,
      "oneClick": false,
      "perMachine": true
    }
  }
}
```

#### 验证

构建完成后，在 `release/` 目录中找到：

- `鱿郁仔仔 Setup X.X.X.exe` - 安装程序
- `鱿郁仔仔 X.X.X.exe` - 免安装版本（如果配置了 portable）
- `checksums.sha256` - 校验和文件

## 常见问题

### Q: PyInstaller 打包后程序很大

**A**: 使用以下优化：

1. 在 `build.spec` 中添加 `excludes`：
```python
excludes=[
    'tkinter',
    'matplotlib',
    'numpy',
    'pandas',
    'scipy',
]
```

2. 启用 UPX 压缩：
```python
exe = EXE(
    # ...
    upx=True,
    upx_exclude=[],
)
```

### Q: electron-builder 打包失败

**A**: 检查以下几点：

1. 确保 `main` 字段指向正确的入口文件
2. 确保 `files` 配置包含所有必要文件
3. 检查 `extraResources` 中的路径是否正确

### Q: 安装包被杀毒软件拦截

**A**: 

1. 使用代码签名证书签名
2. 提交到杀毒软件厂商误报申诉
3. 使用企业签名或购买商业证书

### Q: 如何减小安装包体积

**A**: 

1. 排除不必要的依赖
2. 使用 `--onefile` 而不是 `--onedir`
3. 压缩资源文件
4. 考虑首次启动时下载 Playwright 浏览器

## 高级配置

### 代码签名

```powershell
# 使用 signtool 签名
signtool sign /f certificate.pfx /p password /t http://timestamp.digicert.com "release\*.exe"
```

### 自动更新

配置 electron-updater：

```json
{
  "build": {
    "publish": [{
      "provider": "github",
      "owner": "your-username",
      "repo": "your-repo"
    }]
  }
}
```

### CI/CD 集成

GitHub Actions 示例：

```yaml
name: Build
on:
  push:
    tags: ['v*']

jobs:
  build:
    runs-on: windows-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-node@v3
      - uses: actions/setup-python@v4
      
      - name: Install dependencies
        run: |
          pip install pyinstaller
          cd pc-client/frontend
          npm install
      
      - name: Build
        run: .\scripts\build.ps1
      
      - name: Upload artifacts
        uses: actions/upload-artifact@v3
        with:
          name: installer
          path: pc-client/frontend/release/*.exe
```
