## 待开发功能（按优先级排序）

### P0 — 必须做
1. **批量安装**：多行游戏名输入 → 逐行解析 → 队列管理 → 逐个下载安装
2. **实时安装进度**：进度条 + 当前文件名 + 传输速度 + 剩余时间（ETA）
3. **PushDeer 消息推送**：单游戏完成/失败通知 + 全部完成统一提醒

### P1 — 应该做
4. **自动更新已装游戏**：对比已装版本 vs 网站最新版本 → 自动下载更新
5. **大气层一键升级**：TF卡检测 → 清旧文件(保留 nintendo/emummc) → NAS拉取 → 解压部署
6. **多 Switch 并行安装**：多设备检测 → 独立队列 → 并行安装

### P2 — 可选
7. **云端代码动态加载**：VIP 功能代码存 Cloudflare R2，激活后拉取运行（防反编译）

## 开发规则
1. 编码前思考：先读现有代码理解架构再动手
2. 简洁至上：复用现有模块，最小改动
3. 精准修改：只改必要文件
4. 目标驱动：功能可跑通为验收

## 注意事项
- MTP 传输是串行的，并行安装需要为每台 Switch 维护独立队列
- 实时进度需要 MTP 传输层提供回调（当前 CopyHere 没有真实进度）
- PushDeer Key 由用户在设置页配置
- 大气层升级有风险：必须保留 nintendo/ 和 emummc/ 目录
- 云端代码加载需要 R2 存储，可能涉及额外费用
