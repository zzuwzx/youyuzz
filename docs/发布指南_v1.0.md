# 鱿郁仔仔 v1.0 发布指南

> 版本：v1.0 | 更新日期：2026-05-29

---

## 发布流程

### 1. 准备工作

**检查清单**：
- [ ] 所有测试通过
- [ ] 版本号已更新（`config.py` + `package.json`）
- [ ] CHANGELOG 已更新
- [ ] 图标文件就绪（`assets/icon.ico`）

**更新版本号**：
```bash
# Python 后端
# pc-client/backend/config.py → VERSION = "1.0.0"

# Electron 前端
cd pc-client/frontend
npm version 1.0.0
```

### 2. 本地构建测试

```powershell
# 运行构建脚本
.\scripts\build.ps1 -Clean

# 验证产物
# - pc-client\backend\dist\backend.exe
# - pc-client\frontend\release\鱿郁仔仔 Setup 1.0.0.exe
```

### 3. 提交并打标签

```bash
git add .
git commit -m "release: v1.0.0"
git tag -a v1.0.0 -m "Release v1.0.0"
git push origin master --tags
```

### 4. 自动发布（GitHub Actions）

推送标签后，GitHub Actions 会自动：
1. 运行后端测试
2. 运行前端测试
3. 构建打包
4. 生成校验和
5. 创建 GitHub Release
6. 上传安装包

### 5. 手动发布（备用）

如果自动发布失败：

```powershell
# 1. 本地构建
.\scripts\build.ps1

# 2. 生成校验和
cd pc-client\frontend\release
Get-ChildItem -Filter "*.exe" | ForEach-Object {
    $hash = (Get-FileHash $_.FullName -Algorithm SHA256).Hash
    "$hash  $($_.Name)" | Out-File -FilePath "checksums.sha256" -Append -Encoding UTF8
}

# 3. 创建 GitHub Release
gh release create v1.0.0 `
    --title "鱿郁仔仔 v1.0.0" `
    --notes-file CHANGELOG.md `
    pc-client\frontend\release\*.exe `
    pc-client\frontend\release\checksums.sha256
```

### 6. 部署 Cloudflare CDN

```bash
cd cloud/cdn
npm install
npm run deploy
```

### 7. 更新版本 API

```bash
cd cloud/worker
# 更新 wrangler.toml 中的 APP_VERSION
wrangler secret put APP_VERSION
# 输入: 1.0.0

wrangler secret put APP_DOWNLOAD_URL
# 输入: https://your-cdn.workers.dev/download/1.0.0/鱿郁仔仔 Setup 1.0.0.exe
```

## 回滚流程

如果发布后发现问题：

### 1. 标记为预发布

```bash
gh release edit v1.0.0 --prerelease
```

### 2. 重新发布旧版本

```bash
# 假设要回滚到 v0.9.0
gh release create v1.0.1 \
    --title "鱿郁仔仔 v1.0.1 (Hotfix)" \
    --notes "修复 v1.0.0 的问题" \
    --latest \
    pc-client\frontend\release\*.exe
```

### 3. 更新 CDN

```bash
# 更新 APP_VERSION 为新版本
wrangler secret put APP_VERSION
# 输入: 1.0.1
```

## 常见问题

### Q: GitHub Actions 构建失败？

检查：
1. Python/Node 版本是否匹配
2. 依赖是否完整
3. 图标文件是否存在

### Q: 安装包被杀毒软件误报？

解决方案：
1. 提交到杀毒软件厂商白名单
2. 使用代码签名证书
3. 在发布说明中提醒用户

### Q: 自动更新不工作？

检查：
1. Cloudflare Worker 是否部署
2. `APP_VERSION` 是否更新
3. 网络连接是否正常
