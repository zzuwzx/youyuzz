## Phase 1 — Python核心服务 MVP

**目标**：命令行可完成「输入游戏名 → 爬取 → 网盘下载 → MTP安装」全流程

### 1.1 网站爬虫
- [ ] Playwright 环境搭建（浏览器自动安装脚本）
- [ ] nsthwj.cn 搜索页解析：输入关键词 → 返回结果列表
- [ ] 详情页解析：游戏名、版本号、网盘跳转链接
- [ ] Cookie 持久化存储与复用
- [ ] 反爬处理：UA伪装、headless检测绕过
- [ ] 游戏名称模糊匹配词典（初版手动维护50个热门游戏别名）

**验收**：`python -m backend.scraper search "塞尔达"` 返回正确结果列表

### 1.2 夸克网盘
- [ ] Cookie登录流程 → 持久化存储
- [ ] 网盘文件转存到用户指定目录
- [ ] 获取文件下载直链
- [ ] 断点续传下载（大文件支持）
- [ ] 下载进度回调接口

**验收**：`python -m backend.cloud_disk download <文件ID>` 完成下载

### 1.3 MTP传输
- [ ] 设备检测：扫描MTP设备，筛选出Switch/DBI
- [ ] DBI目录结构识别：定位第5分区(TF卡)和第6分区(NAND)
- [ ] 存储空间查询：TF卡和NAND剩余空间
- [ ] 单文件传输 + 进度回调
- [ ] 安全断开连接

**验收**：`python -m backend.mtp install <本地文件夹>` 完成MTP传输

### 1.4 游戏文件识别
- [ ] 扫描目录，按文件后缀分类（.nsp/.nsz=本体, .nsp/.nsz_upd=更新, .nsp/.nsz_dlc=DLC, .zip=金手指）
- [ ] 支持「规范分类」和「单文件」两种目录形态
- [ ] 按 本体→更新→DLC→金手指 顺序排列

**验收**：任意给定的游戏资源文件夹，正确输出分类结果

### 1.5 缓存管理
- [ ] 下载完成后写入本地缓存目录
- [ ] 1天TTL管理（基于文件修改时间）
- [ ] 安装前优先检查缓存
- [ ] 缓存空间监控和自动清理

### 1.6 本地HTTP API
- [ ] FastAPI服务搭建（localhost:18888）
- [ ] 注册所有路由（搜索、安装、设备、设置、系统）
- [ ] CORS配置（允许Electron本地请求）
- [ ] 统一错误处理和日志

**Phase 1 总验收**：
- 命令行可以完成一次完整的「搜索→下载→MTP安装」
- 所有核心模块有对应的单元测试
- FastAPI服务启动后，`/api/health` 返回 ok

## Phase 2 — Electron GUI 基础版

**目标**：用户通过图形界面完成单游戏自动化安装全流程

### 2.1 项目搭建
- [ ] Vite + React + TypeScript 脚手架
- [ ] Electron 主进程配置（窗口管理、Python子进程管理）
- [ ] Tailwind CSS 集成
- [ ] 基础路由（主页面、设置页、VIP升级页）

### 2.2 基础版UI页面
- [ ] 主页面：搜索输入框 + 搜索按钮 + 结果列表 + 安装按钮 + 状态栏
- [ ] 设备状态指示器（Switch连接状态、TF卡状态）
- [ ] 设置页面：Cookie配置、PushDeer Key、网盘账号
- [ ] VIP升级入口（醒目但不突兀）
- [ ] 加载状态、错误提示、空状态

### 2.3 核心交互
- [ ] 搜索：输入框回车/点击搜索 → 调后端API → 展示结果列表
- [ ] 安装：选择游戏 → 调后端API → 轮询进度 → 实时更新状态
- [ ] 设备检测：启动时自动检测 → 未连接弹窗提醒
- [ ] 本地离线安装：文件夹选择对话框 → 调后端API
- [ ] 设置持久化：所有配置写本地JSON文件

### 2.4 进程管理
- [ ] Electron启动时自动拉起Python后端（子进程）
- [ ] 健康检查：启动后轮询 `/api/health` 直到就绪
- [ ] 关闭时优雅退出Python子进程
- [ ] 异常恢复：Python崩溃时自动重启

### 2.5 打包
- [ ] PyInstaller 打包 Python 后端为 exe（含 libmtp DLL）
- [ ] electron-builder 打包前端 + 嵌入 Python exe
- [ ] Windows 安装包生成（.exe / .msi）

**验收**：
- GUI 完成单游戏自动化安装全流程
- 所有交互有正常的 loading/empty/error 状态
- 打包后的安装包可在另一台 Windows 电脑上安装运行

## Phase 3 — 云端授权体系

**目标**：完整的激活码生成、验证、管理闭环

### 3.1 D1 数据库
- [ ] 执行建表SQL（activation_codes / licenses / admin_users / audit_log）
- [ ] 初始化管理员账号

### 3.2 Workers API
- [ ] `/api/auth/activate` — 激活码验证 + 设备绑定
- [ ] `/api/auth/verify` — 授权有效性验证（每次客户端操作前调用）
- [ ] `/api/auth/heartbeat` — 心跳上报（定期上报，超30天未上报则自动吊销）
- [ ] `/api/version/latest` — 最新版本查询

### 3.3 Pages 管理后台
- [ ] 管理员登录页
- [ ] 兑换码生成页：设置有效期、设备限制、批量数量
- [ ] 用户列表页：查看激活用户、到期时间、状态
- [ ] 用户详情页：禁用/启用、查看操作日志
- [ ] 统计概览页：总用户、活跃用户、今日新增

### 3.4 PC客户端集成
- [ ] 启动时检查授权状态
- [ ] 激活码输入对话框
- [ ] 基础版/VIP功能开关（根据license type）
- [ ] 授权到期提醒

**验收**：
- 管理后台可生成兑换码
- PC客户端输入兑换码后激活VIP功能
- 过期后VIP功能自动锁定
- 修改系统时间不影响授权（云端时间戳校验）
