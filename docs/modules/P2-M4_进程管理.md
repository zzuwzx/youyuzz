# P2-M4: 进程管理

> pc-client/frontend/electron/ | ✅ 已完成

## 目标

Electron 主进程中管理 Python 后端的完整生命周期。

## 功能

- ✅ 启动时自动拉起 Python FastAPI 子进程
- ✅ 健康检查：轮询 /api/health 直到就绪
- ✅ 窗口显示时机：等后端就绪后加载页面并显示窗口
- ✅ 异常恢复：Python 崩溃时自动重新拉起（最多 3 次，间隔 2s）
- ✅ 关闭时优雅退出：app quit → kill python（SIGTERM → 5s 超时 → SIGKILL）
- ✅ 日志捕获：stdout/stderr 写入文件 + IPC 转发到前端 console

## 文件结构

```
electron/
├── main.ts           # 窗口创建 + Python 进程管理
├── pythonBridge.ts   # Python 子进程封装（spawn/stop/events）
└── preload.ts        # 预加载脚本（contextBridge API）
```

## 架构说明

### PythonBridge (`pythonBridge.ts`)

- 封装 `child_process.spawn`，启动 `backend/venv/Scripts/python.exe main.py`
- 设置 `PYTHONUNBUFFERED=1` 确保实时日志输出
- EventEmitter 模式：emit `log` / `error` / `exit` 事件
- `stop()` 方法：SIGTERM 优雅停止，5s 超时后 SIGKILL

### 主进程 (`main.ts`)

- `createWindow()`: 1200×800 窗口，暗色背景 #1A1A2E
- `startPython()`: 初始化 PythonBridge，绑定事件 → 写日志文件 + IPC 转发
- `waitForHealthy()`: 每 500ms 轮询 `http://127.0.0.1:18888/api/health`，最多等 30s
- 健康检查通过 → `webContents.send('python:ready')` → 加载页面 → 显示窗口
- Python 异常退出 → 延迟 2s 自动重启，最多 3 次
- `before-quit` → stop Python → logStream.end() → app.quit()

### 预加载 (`preload.ts`)

- `contextBridge.exposeInMainWorld('electronAPI', {...})`
- 暴露接口：`onPythonLog` / `onPythonReady` / `onPythonError` / `removeAllListeners`

## 前端集成

渲染进程通过 `window.electronAPI` 访问：

```typescript
// 监听 Python 日志（自动转发到 console）
window.electronAPI?.onPythonLog((msg) => console.log('[Python]', msg));

// 监听后端就绪
window.electronAPI?.onPythonReady(() => {
  console.log('Python backend ready');
  // 初始化 API 连接等
});

// 监听错误
window.electronAPI?.onPythonError((msg) => {
  console.error('[Python Error]', msg);
});
```

类型声明位于 `src/electron.d.ts`，无需额外 import。

## 日志

- 文件路径：`pc-client/frontend/logs/python-YYYY-MM-DD.log`
- 保留策略：按日期自动生成，手动清理
- 前端展示：通过 IPC 实时转发到渲染进程 console
